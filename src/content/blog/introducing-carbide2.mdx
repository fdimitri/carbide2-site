## Why this exists
### Claude Sonnet says: 
For the last decade, "cloud IDE" has mostly meant *hosted SaaS that happens to
run an editor*. That's fine, but it skips the part that matters: when the
editor, the terminal, and the filesystem all live on the server, every
collaborator sees the same thing at the same time. No "pull my branch", no
"can you screen-share?", no Slack tab on a second monitor.

CARB/IDE2 is the opposite of hosted SaaS. It's a self-hostable Rails 8.1 app
under GPLv3 that you run on your own box. The browser is a thin client — every
keystroke in a terminal, every file write, every chat message is a server-side
event, broadcast to everyone in the room.

### And to that, I say
Good Lord, Claude! You can make some silly things up. I wrote the original version
over 10 years ago with a friend, and it was supposed to be hacked together and
quick to play with an online-editor. Because while at the time there was Atom and
pair programming, there wasn't a good *multi-user editor with shared terminals*.

We built something that wasn't great in our spare time, but it was interesting.

This is a re-make done with serious LLM assisting (with the benefit of refactoring 
some original stuff) in, as of June 02, 2026 -- approximately 4-5 days. It exists
because I wanted to see how the whole process would go, and to make a slightly better
version of what we already made.

<Callout tone="info">
This is a tech demo, at best. Don't put production data in it yet.
</Callout>

## What's in v0

- A browser-side editor wired to a server-side filesystem in Postgres.
- Real PTYs on the server, streamed to every connected viewer.
- An IRC-style chat with persistent history, per project.
- Git integration (clone / branch / commit / push) inside the workspace.
- A Helm chart for k3d/k3s and a Docker Compose setup for the lazy.
- Super basic tools for an LLM agent to call, where I run a Qwen3 9B on a 5080

## What's next

WebRTC for voice + video, a proper docs site, plugin hooks for language
servers, and a public sandbox-only demo.
